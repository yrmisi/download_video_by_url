# MediaGrab

MediaGrab — это мощный backend-сервис на базе FastAPI, предназначенный для скачивания видео и аудио из различных источников (YouTube и др.) с использованием библиотеки `yt-dlp`.

## Основные возможности

* **Множество профилей загрузки:**
    * `PC/TV`: Максимальное качество (4K/HDR, кодеки VP9/AV1) в контейнере MKV.
    * `Legacy TV`: Совместимость со старыми ТВ (H.264, MP4, звук AAC).
    * `Mobile`: Оптимизированное видео (720p, MP4).
    * `Audio`: Конвертация в MP3 (192kbps) с сохранением метаданных и обложек.
* **Асинхронная обработка:** Загрузка выполняется в фоновом режиме, не блокируя API.
* **Отслеживание статуса:** Реальный прогресс загрузки передается через Redis.
* **Умная очистка:** Автоматическое удаление временных файлов (`.part`, `.ytdl`) и старых загрузок через определенные интервалы времени.
* **История загрузок:** Сохранение метаданных (название, продолжительность, разрешение) в базе данных с кэшированием через Redis.
* **Поддержка прокси:** Настройка SOCKS5 прокси для обхода ограничений.

## Технологический стек

* **Framework:** FastAPI
* **Media Engine:** yt-dlp + FFmpeg
* **Database:** SQLAlchemy (Async)
* **Cache/Task Tracking:** Redis
* **Configuration:** Pydantic Settings

## Структура проекта

```text
app/
├── config/          # Настройки приложения, БД и Redis
├── core/            # Ядро: интеграция с yt-dlp и реестр активных задач
├── database/        # Модели БД и конфигурация сессий
├── routers/         # API эндпоинты (загрузка, история, файлы)
├── services/        # Бизнес-логика (worker, очистка, извлечение инфо)
├── schemas/         # Pydantic модели (DTO)
├── utils/           # Вспомогательные функции (прокси, форматирование имен)
└── main.py          # Точка входа и жизненный цикл приложения
```

## Начало работы

### Предварительные требования

1. Python 3.10+
2. Установленный FFmpeg (обязательно для склейки видео и аудио)
3. Запущенный Redis
4. PostgreSQL (или другая БД, поддерживаемая SQLAlchemy)

### Установка (с использованием [uv](https://github.com/astral-sh/uv))

1. Клонируйте репозиторий:

   ```bash
   git clone <repository_url>
   cd download_video_by_url
   ```

2. Установите зависимости и создайте окружение:

   ```bash
   uv sync
   ```

3. Настройте переменные окружения в файле `.env.app-prod` (путь настраивается в `app/config/app.py`).

### Запуск

```bash
docker compose up --build
```

## API Эндпоинты (Основные)

* `POST /api/extract-info`: Получение информации о видео (название, превью) без скачивания.
* `POST /api/download`: Запуск процесса загрузки. Возвращает `task_id`.
* `GET /api/status/{task_id}`: Получение текущего статуса и прогресса загрузки.
* `GET /api/files/{task_id}`: Скачивание готового файла с "красивым" именем.
* `GET /api/recent-history/{user_id}`: История последних загрузок пользователя.
* `POST /api/cancel/{task_id}`: Принудительная остановка загрузки и очистка файлов.
